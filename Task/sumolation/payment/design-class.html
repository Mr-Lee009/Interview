Sơ đồ riêng cho Callback Flow

1. Provider callback

MoMo, VNPAY hoặc ZaloPay gửi callback/IPN về đúng endpoint /api/payments/callback/{provider}.

2. PaymentController

Controller duy nhất nhận headers, query params, body và chuyển cho service.

3. PaymentService

Service xác định provider, đọc config, lấy adapter từ resolver và điều phối toàn bộ bước verify.

4. Gateway adapter

Adapter parse payload, verify chữ ký, chuẩn hóa trạng thái và trả về kết quả callback chung.

5. Update DB

PaymentService đối chiếu amount, tìm transaction theo provider + gatewayTxnRef rồi cập nhật SUCCESS hoặc FAILED.

Log / design rule

Rule: PaymentService trực tiếp đọc PaymentGatewayConfigRepository, chọn adapter qua PaymentGatewayResolver, và xử lý callback cập nhật trạng thái DB. Không cần tách thêm service nhỏ nếu chưa tái sử dụng được.
