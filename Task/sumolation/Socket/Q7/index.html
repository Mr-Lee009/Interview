Mô phỏng xác thực WebSocket

Mỗi loại xác thực đều đi qua cùng một ý tưởng: client gửi thông tin xác thực trong lúc bắt tay, server kiểm tra rồi quyết định có cho phép `Upgrade` sang WebSocket hay không.

Client

Browser, SPA, mobile app hoặc desktop app mở kết nối WebSocket.

Handshake

Yêu cầu ban đầu đi qua HTTP để xin `Upgrade: websocket`.

Server

Kiểm tra cookie hoặc token rồi quyết định cho phép hay từ chối kết nối.
