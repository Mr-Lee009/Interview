Kiến trúc Chat WebSocket cho 100.000 kết nối

Một sơ đồ cho ba câu hỏi phỏng vấn thường đi cùng nhau: scale số lượng connection lớn, đồng bộ message giữa nhiều instance Spring Boot, và không làm mất message khi user offline.

Clients

Nhiều mobile app, browser tab, desktop app cùng mở WebSocket.

Load Balancer

Phân phối kết nối tới nhiều Spring Boot WebSocket nodes.

Spring Node 1

Giữ một phần session và push realtime cho user local.

Spring Node 2

Không biết session của node khác nếu chỉ nhìn local memory.

Spring Node 3

Có thể là node đang giữ người nhận hoặc xử lý reconnect.

Redis Presence

Biết user đang online ở node nào.

Kafka / Redis PubSub

Đồng bộ event giữa các instance và fan-out message.

DB / Message Store

Lưu message bền vững để replay khi user offline.

User Offline

Không nhận realtime, sẽ lấy bù khi reconnect.
