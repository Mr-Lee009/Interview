# Behavioral Patterns trong Java

Thư mục này tách riêng 11 `Behavioral Patterns`, mỗi file gồm:

- Định nghĩa pattern
- Cách dùng
- Khi nào nên dùng
- Code Java mẫu
- Comment chi tiết ngay trong code

## Tóm tắt nhanh

1. `Chain of Responsibility`: đẩy request qua chuỗi handler.
2. `Command`: đóng gói thao tác thành object.
3. `Interpreter`: diễn giải biểu thức hoặc ngôn ngữ đơn giản.
4. `Iterator`: duyệt collection mà không lộ cấu trúc bên trong.
5. `Mediator`: gom giao tiếp nhiều object về một đầu mối.
6. `Memento`: lưu và phục hồi trạng thái object.
7. `Observer`: tự động thông báo khi state thay đổi.
8. `State`: thay đổi hành vi theo trạng thái hiện tại.
9. `Strategy`: thay thế thuật toán linh hoạt.
10. `Template Method`: cố định skeleton thuật toán, để subclass cài các bước.
11. `Visitor`: thêm hành vi mới mà không sửa class phần tử.
