# Structural Patterns trong Java

Thư mục này tách riêng 7 `Structural Patterns`, mỗi file gồm:

- Định nghĩa pattern
- Cách dùng
- Khi nào nên dùng
- Code Java mẫu
- Comment chi tiết ngay trong code

## Danh sách file

1. `adapter.md`
2. `bridge.md`
3. `composite.md`
4. `decorator.md`
5. `facade.md`
6. `flyweight.md`
7. `proxy.md`

## Tóm tắt nhanh

1. `Adapter`: chuyển đổi interface để các class tương thích với nhau.
2. `Bridge`: tách abstraction khỏi implementation để thay đổi độc lập.
3. `Composite`: biểu diễn cấu trúc cây giữa object đơn và object nhóm.
4. `Decorator`: bọc object để thêm chức năng động.
5. `Facade`: cung cấp interface đơn giản cho hệ thống phức tạp.
6. `Flyweight`: chia sẻ state dùng chung để tiết kiệm bộ nhớ.
7. `Proxy`: tạo object đại diện để kiểm soát truy cập object thật.
